Header navigation (no Bootstrap)

Mobile: Menu row stays put; tab list opens as an overlay under it. Submenus expand in-flow inside that list. Desktop: tabs always visible; submenus sit under each tab. Open via hover (fine pointer) or .is-open / .is-menu-open on tap.

Canonical patterns (backend recipes)

Three tab kinds. Backend should pick one recipe per nav page based on SubPagesExist and child count.

A. Plain link (no children)

  • <li id="nav_tab_*" class="relative flex_auto m_0 p_0 flex flex_column"> — no nav-has-submenu
  • Trigger row: <div class="nav-tab-trigger flex flex_row items_center flex_auto w_100">
  • One <a class="nav-link-arches flex_70 flex_auto nav-submenu-label-arches nowrap p-x_3 p-y_3 a:bg_accent-n3 h:bg_accent-n1 c_white not-link block h:undecorated">
  • Current page: add active on <li> + aria-current="page" on the link

B. Dropdown — 1 column (1–5 children)

  • <li id="nav_tab_*" class="relative flex_auto nav-has-submenu m_0 p_0 flex flex_column">
  • Trigger row: <div class="nav-tab-trigger flex flex_row items_center flex_auto w_100">
  • Toggle <button class="… nav-submenu-toggle … display_none:md block:touch …" aria-controls="nav_menu_*"> — tap toggles .is-open on the li
  • Label <a class="… nav-submenu-label-arches …"> — no interestfor
  • Panel sibling (in flow, no popover): class="nav-submenu-popover dropdown-menu-arches m-b_3 bg_white …"
  • Inner grid: grid columns_1 p-t_2 p-b_3 p-x_2p-x_3:md
  • One .dropdown_column → child list + link recipe unchanged

C. Dropdown — 2 columns (> 5 children) — additive on B

  • Why: long child lists need a wider panel and a visual split
  • Panel adds w_auto:md
  • Inner grid adds columns_2:md gap-x_3
  • Split children into two .dropdown_column blocks
  • 2nd column only: br-l_1 br_solid:md br_none br_black-2 p-l_3:md

Shared behavior

  • Mobile Menu: toggle adds .is-menu-open on #header_nav-arches. Panel is position: absolute; top: 100% under the toggle row — Menu does not move.
  • Submenus (mobile): position: static inside that panel — push following tabs
  • Submenus (desktop): anchor to .nav-tab-trigger bottom (main nav bar edge) + position-try-fallbacks; m-x_3 zeroed so it doesn’t skew the box
  • Open (fine pointer): :hover / :focus-within on .nav-has-submenu
  • Open (tap): .is-open on tab li; .is-menu-open for Menu
  • No popover on submenus or the mobile panel — popover top-layer fought layout

Resolved decisions

  1. Dropped Popover API for nav panels — class toggles + CSS positioning only
  2. #header_nav_toggle_row is the only in-flow mobile chrome; list overlays below it
  3. .nav-tab-trigger + li.flex_column for submenu stacking inside the list
  4. Hover gated by (hover: hover) and (pointer: fine)
