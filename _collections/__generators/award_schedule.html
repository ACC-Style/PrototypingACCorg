Paste date list CSV

Generic date-oriented list pattern. The CMS copy is a single flat data-role="source" list — web authors paste items in any order and add new rows during the year without sorting. Client-side JavaScript builds Open, Upcoming, and Past sections at runtime based on the visitor's date.

Expected columns

  1. title
  2. start_date (MM-DD, evergreen — current year applied at runtime)
  3. end_date (MM-DD, optional)
  4. city (optional)
  5. country (optional)
  6. presenter (optional)
  7. flags — comma-separated (e.g. FIT, International)
  8. description
  9. learn_more_url — date badge link; also Learn More button when open
  10. action_url — Apply Now button (open items only)

Rows in the flat source list are ordered alphabetically by title (leading The and ACC are ignored). At runtime, Open awards sort alphabetically; Upcoming and Past sort chronologically, then alphabetically by title. Item metadata displays as: date | location (if present) | presenter (if present). Section headings and labels are configurable on the root data-component="DateList" element. The date badge always links to learn_more_url. Apply Now and Learn More footer buttons appear only when an item is open. Paste the flat source list into the CMS, then add the companion CMS script below it. The script builds Open, Upcoming, and Past sections at runtime, sorts by the visitor's date, updates card visuals, and removes empty sections from the DOM. To add items later, append new <li> cards inside data-role="source" in HTML source view — no manual sorting required.

Copy both blocks into the CMS: the flat data-role="source" list first, then the self-contained CMS runtime script below it. The script is a standalone IIFE for CMS paste only. The preview pane uses the same logic via the generator's embedded preview driver — not the copy string in this textarea.
